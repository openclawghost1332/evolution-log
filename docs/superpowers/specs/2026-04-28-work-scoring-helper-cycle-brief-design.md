# Design: Work Scoring Helper Cycle Brief Export

Generated by brainstorming on 2026-04-28
Status: APPROVED
Mode: Builder

## Problem Statement

The existing work scoring helper ranks ideas and can export/import saved ideas, but it still leaves a gap between choosing an idea and turning that choice into a usable cycle artifact. After scoring, the operator or future cycle agent still has to manually rewrite the selected idea into a concise markdown brief for cycle notes, handoffs, or issue comments.

## What Makes This Cool

The tool should not only help decide what to build, it should help immediately launch the work. A single click should turn the highest-confidence idea into a compact markdown brief with score, rubric breakdown, section context, and notes, ready to paste into a cycle note, PR description, or planning issue.

## Premises

1. The best next improvement is one that shortens the path from ranking to shipping, not another scoring tweak.
2. The helper should stay static, browser-only, and dependency-free.
3. Exporting a markdown brief is more useful for the lab workflow than adding another persistence format.

## Approaches Considered

### Approach A: Saved-idea markdown export only
- Summary: Add a button on each saved idea to generate a markdown cycle brief and copy it to the clipboard.
- Effort: S
- Risk: Low
- Pros:
  - Smallest diff
  - Directly improves post-scoring workflow
  - Easy to test with pure helper functions
- Cons:
  - Does not help before an idea is saved
  - Clipboard UX needs a fallback for unsupported browsers
- Reuses: existing saved-idea cards, scoring breakdown data

### Approach B: Candidate and saved-idea brief generation
- Summary: Add shared markdown brief generation logic for both parsed candidates and saved ideas, with UI buttons to preview/copy the brief.
- Effort: M
- Risk: Low
- Pros:
  - Covers both pre-save and post-save workflows
  - Central pure function can be tested in Node
  - Makes the tool more obviously useful during cycle selection
- Cons:
  - Slightly larger UI change
  - Needs normalization for candidate vs saved-idea shapes
- Reuses: existing candidate ranking, saved idea rendering, shared scoring module

### Approach C: Full cycle-note scaffold generator
- Summary: Generate a larger markdown document with sections for summary, artifact, tests, blockers, and next steps.
- Effort: M
- Risk: Medium
- Pros:
  - Strongest handoff artifact
  - Could align directly with cycle-note templates
- Cons:
  - More opinionated than necessary
  - Harder to keep concise and generally useful
  - Risks bloating a lightweight browser tool
- Reuses: scoring metadata, note text, timestamp

## Recommended Approach

Choose Approach B because it gives the fastest practical jump from backlog ranking to execution while keeping the implementation small, testable, and reusable.

## Design

### Shared logic
Add a pure formatter in `previews/work-scoring-helper/scoring.js` that accepts an idea-like object plus rubric breakdown metadata and returns markdown with:
- title
- optional section
- total score
- bullet list of rubric values
- notes/rationale
- exported timestamp line

Also add a small helper that converts imported candidates into score-ready brief input using the same estimated scoring path already used for ranking.

### Saved ideas UX
Each saved idea card gets a `Copy brief` button. Clicking it copies the generated markdown to the clipboard and updates the existing transfer status line with a success or failure message.

### Candidate UX
Each parsed candidate card gets a `Copy brief` button next to `Use in scorer`. The candidate brief uses estimated score plus preset context so a cycle can move from importer to action without first saving the idea.

### Browser fallback
Use `navigator.clipboard.writeText` when available. If clipboard write fails or is unavailable, fall back to a temporary textarea plus `document.execCommand('copy')`. Keep the implementation local to the page script.

### Testing
Add Node tests for:
- markdown brief formatting for saved ideas
- candidate brief generation from estimated preset scores
- output including section, score, notes, and rubric bullets in a stable format
