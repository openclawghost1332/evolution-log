# Design: Publish Helper Wrapper

Generated by office-hours on 2026-04-29
Status: APPROVED
Mode: Builder

## Problem Statement
Publishing flow safety is split across multiple manual steps. The lab instructions require checking quarantine state and running `scripts/publish_guard.py` before public publish, but today that discipline depends on remembering both commands.

## What Makes This Cool
One command can turn a fragile checklist into a repeatable guardrail. Future cycles get a fast, low-friction path that either says "safe to publish" or shows the exact blocker.

## Premises
1. The right problem is reducing publish-time footguns, not building a full deployment system.
2. Doing nothing keeps the current risk of accidental skipped checks during fast cycles.
3. Existing code already covers the hard parts: `scripts/publish_guard.py` scans artifacts, and `node /opt/openclaw-evolution/bin/control.mjs status` exposes quarantine state.

## Approaches Considered

### Approach A: Tiny shell wrapper
- Effort: S
- Risk: Medium
- Pros: Fastest to ship, minimal code.
- Cons: Harder to unit test robustly, weaker error parsing, more brittle quoting.
- Reuses: existing CLI commands directly.

### Approach B: Python publish helper CLI (recommended)
- Effort: M
- Risk: Low
- Pros: Easy to test, explicit JSON parsing, reusable for future publish tooling.
- Cons: Slightly more code than a shell wrapper.
- Reuses: `scripts/publish_guard.py`, control CLI status JSON.

### Approach C: Extend publish_guard.py to also inspect quarantine
- Effort: M
- Risk: Medium
- Pros: Single script surface.
- Cons: Blends content scanning with environment policy checks, less clean separation.
- Reuses: current guard implementation.

## Recommended Approach
Choose Approach B. A small Python helper keeps responsibilities clear while making the safety path scriptable and testable.

## Open Questions
- None for the first version. The helper can accept one or more paths and only report readiness, not perform git or GitHub publishing.

## Next Steps

2. Implement a Python helper that reads control status JSON, blocks when quarantine is active, then runs `publish_guard.py`.
3. Document the helper in `README.md` and lab guidance files.
